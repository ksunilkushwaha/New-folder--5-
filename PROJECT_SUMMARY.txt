DATABASE SCHEMA
===============

Table: records
  id INTEGER PRIMARY KEY AUTOINCREMENT
  date TEXT NOT NULL
  total_earning INTEGER NOT NULL
  total_expenses INTEGER NOT NULL
  net INTEGER NOT NULL
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

Table: earnings
  id INTEGER PRIMARY KEY AUTOINCREMENT
  record_id INTEGER NOT NULL (FOREIGN KEY)
  name TEXT NOT NULL
  amount INTEGER NOT NULL

Table: expenses
  id INTEGER PRIMARY KEY AUTOINCREMENT
  record_id INTEGER NOT NULL (FOREIGN KEY)
  name TEXT NOT NULL
  amount INTEGER NOT NULL

âœ… Normalized design (3NF)
âœ… Referential integrity via foreign keys
âœ… Timestamps for audit trail
âœ… No data duplication


KEY ROUTES
==========

GET /
  - Serves index.html
  - User inputs earnings/expenses
  - Returns: Empty form with 1 default earning + 1 default expense

POST /calculate
  - Processes form submission
  - Parses dynamic form fields (earning_name_*, earning_amount_*)
  - Inserts into DB via save_record_to_db()
  - Exports to CSV via export_to_csv()
  - Returns: results.html with summary

GET /history
  - Queries DB: SELECT * FROM records ORDER BY date DESC
  - Reconstructs records with earnings/expenses via JOINs
  - Passes data to history.html
  - Returns: Table + Chart.js visualizations

NEXT: DEPLOYMENT

To deploy to Render:
